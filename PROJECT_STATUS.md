## Strategic Decision: Phase 3 before Phase 4

**Decision:** Scraper Integration (Phase 3) will be implemented BEFORE the
REST API Layer (Phase 4).

**Reasoning:** The Raw+Parsed Salary Pattern and `existsBySourceUrl`
deduplication logic must be validated against real-world Playwright data
before we build an API on top of it. Building the REST layer on unverified
assumptions would create rework risk.
